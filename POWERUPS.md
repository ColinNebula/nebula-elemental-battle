## ğŸ® Gameplay Integration

### Using Consumables

1. Open Inventory (ğŸ“¦ button in Main Menu)
2. Navigate to "Consumables" tab
3. Select item to view details
4. Click "Use Item" to activate effect
5. Effect applies to next game or current game if in progress

### Equipping Items

1. Open Inventory
2. Navigate to "Equipment" tab
3. View equipped items in slots
4. Select new equipment from available items
5. Click "Equip" to replace current item
6. Equipment bonuses apply immediately

### Adding Rare Cards to Deck

1. Open Inventory
2. Navigate to "Rare Cards" or "Wild Cards" tab
3. Select card to view stats
4. Click "Add to Active Deck"
5. Card becomes available in next game

## ğŸ”§ Technical Details

### Data Persistence

- **LocalStorage**: All inventory data saved locally
- **Auto-Save**: Changes persist immediately
- **Recovery**: Inventory restored on page reload

### Inventory Class

```javascript
PlayerInventory {
  rareCards: []        // Unique collectible cards
  consumables: []      // Stackable items with quantity
```

### Methods

- `addItem(item, quantity)` - Add item to inventory
- `useConsumable(itemId)` - Use consumable and reduce quantity
- `equipItem(item)` - Equip item to slot
- `unequipItem(slot)` - Remove equipped item
- `getEquipmentBonuses()` - Calculate all active bonuses
- `purchase(item)` - Buy item from shop
- `addCurrency(amount)` - Add gold to balance

## ğŸ“ Developer Notes

### Adding New Items

1. Define item in appropriate constant (RARE_CARDS, CONSUMABLES, etc.)
2. Include all properties: id, name, rarity, effect, description, icon
3. Add to loot tables if droppable
4. Implement effect logic in GameClient if needed
5. Test drop rates and balance

### Balancing Guidelines

- **Common**: Basic utility, affordable
- **Uncommon**: Modest advantages, moderate cost
- **Rare**: Significant impact, expensive
- **Epic**: Game-changing effects, very expensive
- **Legendary**: Unique mechanics, premium price

### Performance Considerations

- Inventory limited to reasonable sizes (no hard limit but UI optimized for ~100 items)
- LocalStorage size monitored (under 5MB recommended)
- Equipment bonuses calculated once per change, not per card
- Loot generation is client-side for offline play
